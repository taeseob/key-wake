# Key-Wake (키웨이크) - 맞춤형 알람 해제 앱 기능 요구사항 및 구현 계획

사용자가 설정한 특정 미션(행동 유도, 텍스트 입력, 음성 인식)을 올바르게 수행하기 전까지 절대 멈추지 않는 Android 전용 알람 애플리케이션 **'Key-Wake'**의 개발을 위한 기능 요구사항 정의서 및 기술 구현 계획입니다.

---

## 1. 제품 개념 및 주요 목표
- **제품명**: Key-Wake (키웨이크)
- **핵심 가치**: 단순히 기상하는 것뿐만 아니라, **잠에서 깨어나 꼭 해야 할 특정 행동(자리 이동, 확언 타이핑, 음성 발화 등)을 즉각적으로 유도**하여 생산적인 하루의 시작을 돕습니다.
- **주요 기능**:
  - 다중 알람 설정 및 로컬 관리
  - 알람 해제용 커스텀 미션 (텍스트 타이핑, 걸음 수 측정 이동, 내장 음성 인식)
  - 알람 구동 시 강제 화면 점유 및 볼륨 통제 (진동 모드 지원 및 전화 수신 시 예외 처리 포함)
  - 다채로운 테마/톤앤매너 선택 기능

---

## 2. 기능 요구사항 (Functional Requirements)

### 2.1. 알람 관리 (Alarm Management)
- **다중 알람 등록/편집/삭제**:
  - 알람 시간 설정 (시, 분) 및 요일별 반복 설정.
  - 알람 이름(라벨) 설정 (예: "출근 준비 및 스트레칭").
- **알람 개별 활성화/비활성화**:
  - 토글 스위치 제공.
- **알람 출력 타입 선택 (소리 vs 진동)**:
  - 알람 등록/수정 시 **소리 대신 진동으로만 알람이 울리도록** 선택할 수 있는 옵션을 제공합니다.
  - 진동 패턴 강도 조절 옵션을 포함합니다.
- **알람 옵션 상세 설정**:
  - 알람 소리 선택 (소리 선택 시: 기본 벨소리, 로컬 음원 파일).
  - 강제 최대 볼륨 출력 및 진동 활성화 여부.
  - **스누즈(다시 알림) 기능 제외**: 사용자가 완벽히 깰 때까지 알람을 멈추지 않아야 하므로 임시 중단 기능인 스누즈는 지원하지 않습니다.

### 2.2. 알람 해제 미션 (Wake-up & Action Missions)
사용자는 각 알람마다 아래 미션 중 하나를 지정하여 강제 행동을 유도합니다.

- **텍스트 입력 미션 (Text Mission)**:
  - **작동**: 화면에 제시된 사용자의 커스텀 문장(예: 아침 확언, 오늘 계획)을 타이핑해야 종료됩니다.
  - **검증 규칙**: 띄어쓰기 및 맞춤법 100% 일치.
  - **타이핑 동작**: 입력 중 오타가 발생하면 **백스페이스를 사용하여 자유롭게 수정** 가능합니다.
- **걸음 수 측정 미션 (Step Counter Mission)**:
  - **작동**: 사용자가 사무실 자리나 침대에서 일어나 물리적으로 이동하게 만들기 위해, 알람 작동 후 **설정된 걸음 수(예: 30~50걸음) 이상 걸어야 알람이 종료**되는 미션입니다.
  - **기술 구현**: Android의 `Step Counter` / `Step Detector` 센서를 활용하여 백그라운드 및 포그라운드에서 실시간 걸음 수를 측정합니다. GPS 음영 지역인 실내(사무실 등)에서도 정확하게 위치 이동을 인지할 수 있는 가장 신뢰도 높은 방식입니다.
- **음성 입력 미션 (Voice Mission)**:
  - **작동**: 화면의 마이크 버튼을 누르고 사전에 설정한 문장을 말하면 종료됩니다.
  - **기술 구현**: Android 내장 `SpeechRecognizer` API를 사용합니다.
  - **오프라인 대응 및 안내 시나리오 (중요)**:
    - **설정 단계 안내**: 알람 등록/수정 화면에서 음성 미션을 선택할 때, **오프라인 환경(비행기 모드, 인터넷 불가 상태 등)에서의 인식 제한**에 대해 사용자에게 경고 팝업 또는 안내 문구를 노출합니다.
    - **오프라인 언어팩 안내**: 원활한 오프라인 인식을 위해 Android 시스템 설정에서 **오프라인 한국어 음성 인식 언어팩을 미리 다운로드하도록 유도**하는 안내 링크 또는 가이드를 제공합니다.
    - **오프라인 상태에서의 폴백(Fallback) 작동**: 알람이 울릴 때 네트워크 연결이 불가능하여 음성 인식이 완전히 실패하는 경우, 사용자가 영원히 알람을 끄지 못하는 상황을 방지하기 위해 **기본 알람(사운드/진동 재생 시간과 횟수가 제한되어 자동 종료되는 일반적인 알람) 수준으로 전환되어 동작**하도록 폴백 시나리오를 구성합니다.

### 2.3. 알람 실행 화면 및 예외 처리
- **강제 점유 및 오버레이**:
  - 알람 작동 시 화면을 강제로 켜고 잠금 화면 위로 알람 해제용 미션 화면을 오버레이 표시합니다.
- **전화 수신 시 예외 처리**:
  - 알람이 울리는 도중 시스템에 전화 수신(Incoming Call) 이벤트가 발생할 경우, **통화에 방해가 되지 않도록 알람 음/진동을 즉시 일시적으로 묵음(Mute) 및 정지 처리**합니다. 통화가 종료되면 알람이 다시 복구되어 작동합니다.
- **테마 선택 기능 (Theme Selection)**:
  - 사용자가 설정 화면에서 앱 전체의 디자인 톤앤매너(예: 현대적인 다크 모드, 네온 컬러 하이라이트 테마, 차분한 라이트 모드 등)를 직접 선택할 수 있도록 지원합니다.

## 3. 비기능 및 플랫폼 기술 분석 (Technical & Platform Analysis)

### 3.1. 대상 플랫폼: Android 전용 (Kotlin Native)
- 확실한 백그라운드 정확도 및 하드웨어 제어(잠금화면 오버레이, 무음 모드 오버라이드, 센서 제어)를 보장하기 위해 Android Native 환경으로 제한하여 개발합니다.

### 3.2. 핵심 라이브러리 및 API 사용 계획
1. **음성 인식 (STT)**: Android SDK 내장 `android.speech.SpeechRecognizer`
2. **걸음 수 센서**: `SensorManager` (센서 타입 `Sensor.TYPE_STEP_COUNTER` 및 `Sensor.TYPE_STEP_DETECTOR`)
3. **볼륨 & 진동 제어**: `AudioManager` 및 `VibratorManager` (진동 피드백 및 알람 진동 패턴 제어)
4. **오버레이 화면**: `WindowManager.LayoutParams.TYPE_APPLICATION_OVERLAY` 및 잠금화면 표시 관련 플래그 (`FLAG_SHOW_WHEN_LOCKED`, `FLAG_TURN_SCREEN_ON`)

## 4. 검증 계획 (Verification Plan)

### 수동 검증 시나리오
1. **진동 모드 테스트**: 소리 없이 진동으로만 알람이 정확하게 울리고 해제되는지 검증.
2. **음성 오프라인 폴백 테스트**: 인터넷 연결을 차단한 상태(비행기 모드)에서 음성 알람 작동 시, 인식 실패 상황에서 기본 알람 규격에 따라 일정 재생 횟수/시간 이후 정상 자동 종료 및 경고가 해제되는지 확인.
3. **걸음 수 측정 테스트**: 알람 발생 후 자리에 앉아있을 때는 알람이 멈추지 않고, 걷기 시작하여 설정된 임계치(예: 30걸음)에 도달하는 순간 알람이 정상 해제되는지 검증.
4. **전화 수신 테스트**: 알람 도중 전화 수신 시 진동/소리가 즉시 정지되고, 통화 종료 즉시 알람이 다시 개시되는지 검증.
